JOE ASSISTANT

Backend | Natural Language Processing

" Is there a way to automate people's routine tasks? "

So far, Joe Assistant is a desktop application written in Python which is capable of playing music on Spotify just using your voice.


The idea arose from the need to understand how current voice assistants work and the desire to build one that improves, in some areas, both the performance they offer and people's lives.


As it's an ambitious project, I consider a lot of research is needed in two key aspects: on the one hand, the scalability of the project (by creating a good architecture) and, on the other hand, the Joe's capacity to understand (by using Natural Language Processing). Therefore, I'm currently learning about these fields.

OBJECTIVES

One of the main objectives is to be able to automate any routine task a person may have on a daily basis. Some examples are: playing music, setting reminders, controlling our house lights, searching information on Google and even being able to interact with our WhatsApp account.


In the long term, I'd like the project to be Open Source so that each developer can easily add modules by adding the functionalities they want. Therefore, it is really important to define a clear architecture.

CHALLENGES

The first challenge I faced with this project was to know what was the flow of interaction between a person and the assistant, to look for the necessary tools and to implement it in a simple way to make tests. When I understood the flow, I realized that there were only two ways to interact with Joe: by clicking a button (that it would be a simple but tedious task for the user) or by waking up the assistant with your own voice.


At this point, I decided to implement the first option to check that everything was working properly and then research how could I implement the second one, by developing what is known as "hotword detection".


In that second approach, I was using the Google API to be able to go from voice to text (and vice versa), and that was becoming a problem. Whenever I tried to activate Joe, it was being activated late because everything was being processed on remote servers and the latency was affecting. So, I needed a way to process the user's petition locally.


I came up with two different ideas. On the one hand, trying to create a neural network that was able to detect the word "Joe". And, on the other hand, trying to analyze the audio signal to know if the hotword was activated.


I ended up choosing the second option, since it was the best in terms of performance and development time . Also, I was lucky enough to find a library that made this process much easier, so I only had to learn a few basics about audio signal processing to be able to implement it.
